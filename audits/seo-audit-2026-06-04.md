# SEO Audit — Clearhead — 2026-06-04

## Summary
Site is in strong technical SEO shape: every indexable page has a unique title within length limits, a meta description, canonical, OG/Twitter cards, one H1, and validated structured data. All 14 sitemap URLs return 200, robots.txt blocks utility pages correctly, and image weights are well under thresholds. The most actionable issues are a tiny set of length and consistency drifts (one description 2 chars over the limit, one OG description 16 chars over, one FAQPage question that doesn't match its visible `<summary>` due to a smart-quote/straight-quote mismatch) plus a soft risk: two blog posts and the homepage hero now point to external Unsplash images for their OG/hero, which is fragile.

## Critical (0)

_None._

## High (1)

- **FAQPage rich-result mismatch on `post-lonely.html` — question text doesn't match visible content** — `post-lonely.html:68` (JSON-LD) vs `post-lonely.html:232` (visible) — JSON-LD has `"What does it mean to feel 'unknown'?"` (straight single quotes); visible `<summary>` has `What does it mean to feel "unknown"?` (straight double quotes). Google requires FAQPage `name` to match the visible question verbatim — this can disqualify the entire FAQ rich result for the page. Fix: align both to the same quote style (recommend double quotes in both, matching the visible HTML).

## Medium (3)

- **Meta description over 160-char limit on `career-friction.html`** — `career-friction.html:8` — current length 162 chars; trim to ≤160. Suggested: drop trailing `— and see what staying costs.` or shorten `audit` phrasing.
- **OG description over recommended 160-char limit on `post-ai-loneliness.html`** — `post-ai-loneliness.html:14` — 176 chars; Twitter/LinkedIn will truncate around 160. Trim to ≤160 (e.g. drop the final clause `Only human contact did.`).
- **External Unsplash dependency for OG images on two blog posts** — `post-ai-loneliness.html:18`, `post-new-city-loneliness.html:17` — both use `images.unsplash.com` URLs for `og:image`. If Unsplash rate-limits, blocks hotlinking, or rotates the photo ID, social previews silently break. Fix: download the Unsplash images, store under `~/Documents/Zen/`, and point `og:image` at the local copies (matches the pattern used by all other posts pointing at `VJ.jpg`).

## Low (6)

- **External Unsplash dependency for homepage hero + 3 card images** — `index.html:172, 186, 194, 202` — the local copies (`coaching-early.jpg`, `coaching-mid.jpg`, `coaching-grad.jpg`) still exist in the repo. Local images load faster (no extra DNS+TLS+CDN hop), avoid LCP risk if Unsplash is slow, and aren't subject to Unsplash policy changes. Switch back to the local assets unless there's a specific reason to use Unsplash.
- **`LocalBusiness` JSON-LD missing `image` and `priceRange`** — `index.html:80` — both are recommended by Google for LocalBusiness rich results and help map-pack eligibility. Suggested: `"image": "https://clearhead.in/VJ.jpg"`, `"priceRange": "₹₹"` (or actual rupee range).
- **Sitemap `<lastmod>` values are 4–5 days stale relative to recent local edits** — `sitemap.xml:5` and throughout — newest `lastmod` is `2026-05-31`; today is `2026-06-04`. Not harmful (Google checks ETag), but if any page content was substantively updated since, bumping `lastmod` invites a recrawl.
- **`SoftwareApplication` schema on the 4 tool pages omits `aggregateRating`** — `runway.html`, `bandwidth.html`, `career-friction.html`, `ai-relevance.html` — `aggregateRating` is the field that enables the star-rating rich result. Skip if you don't actually collect ratings (recommended for a calm, mature brand); otherwise add real ratings.
- **`thank-you.html` and `quiz.html` lack `og:image`** — both pages — they're `noindex,nofollow`, so this has no organic-search impact, but if the URLs get shared (e.g., quiz link sent via WhatsApp) the preview will be image-less. Add `og:image` pointing to `VJ.jpg` for parity with the rest of the site.
- **31 unused SVG files in repo root (`peep-*.svg`, `mix-*.svg`, `happy.svg`)** — none are referenced by any HTML page. Not a ranking issue, but they bloat the deploy and slow audits. Move to `_unused/` or delete.
