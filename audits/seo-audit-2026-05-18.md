# SEO Audit — Clearhead — 2026-05-18

## Summary

The site is structurally sound: one clean H1 per page, descriptive alt text on all images, no render-blocking scripts, strong LocalBusiness/Person/FAQPage schema on the homepage, and all images under 500 KB. The most urgent issue is a likely Netlify deployment failure — the latest commit (414fc2d, 16 May) has been pushed to GitHub but appears not to have reached the live server, meaning a broken OG image on the blog page and all the recent schema improvements are absent from what Google is crawling. After resolving that, the highest-leverage fixes are trimming over-length meta descriptions (every content page is well over the 155-char cutoff) and aligning the sitemap and canonical tags with the clean URLs Netlify actually serves.

## Critical (1)

- **Latest commit not deployed — broken OG image on blog page** — `blog.html` (live) still references `https://clearhead.in/VJ.png`; VJ.png was deleted in commit 414fc2d and replaced with VJ.jpg. Social shares of the blog page will show a broken preview image. All SEO improvements from that commit — FAQ and BreadcrumbList schemas on all five blog posts, expanded homepage FAQ schema, loneliness post featured on homepage — are absent from the live site. Check the Netlify dashboard for a failed/stalled deploy triggered from the 16 May push to `origin/main`.

---

## High (3)

- **Sitemap URLs use `.html` extension** — `sitemap.xml` — All inner-page entries use `.html` suffixes (e.g. `https://clearhead.in/blog.html`) but Netlify pretty-URL routing serves and canonicalises these as `/blog`, `/post-ai`, etc. Crawlers will follow the redirect on each URL, wasting crawl budget and creating unnecessary redirect hops. Change all sitemap `<loc>` values to the clean-URL form: `https://clearhead.in/blog`, `https://clearhead.in/post-ai`, and so on.

- **Canonical tags use `.html` extension** — `blog.html`, `post-ai.html`, `post-unheard.html`, `post-lonely.html`, `post-ai-loneliness.html`, `post-conversation.html` — Each page's `<link rel="canonical">` points to its `.html` URL while Netlify serves (and redirects to) the extension-free URL. Google consolidates signals on the canonical; if the canonical differs from the served URL, link equity and index coverage are split. Update each canonical `href` to the clean URL (e.g. `https://clearhead.in/blog`, `https://clearhead.in/post-ai`).

- **Meta descriptions significantly over length** — Multiple pages — Google truncates descriptions at approximately 155 characters; every content page currently exceeds this. The index.html description is 263 chars; all five blog posts run 194–229 chars. The copy that appears in search snippets will be cut mid-sentence. Target 120–155 chars for all pages. Current lengths:
  - `index.html`: 263 chars → trim to ≤155
  - `post-ai.html`: 194 chars → trim to ≤155
  - `post-unheard.html`: 198 chars → trim to ≤155
  - `post-lonely.html`: 200 chars → trim to ≤155
  - `post-ai-loneliness.html`: 229 chars → trim to ≤155
  - `post-conversation.html`: 194 chars → trim to ≤155

## Medium (3)

- **Homepage title too long** — `index.html` — Title is 105 characters. Google displays approximately 55–60 characters in desktop SERPs, so `…Counselling for AI Anxiety, Mid-Career Identity & Loneliness` is truncated. Consider: `Clearhead Mumbai | Vaibhav Jain — Coach & Counsellor` (52 chars) with the keyword detail moved to the description. The current title is strong but the tail gets cut before the audience and specialty descriptors render.

- **Article schema `dateModified` stale on four posts** — `post-ai.html`, `post-unheard.html`, `post-lonely.html`, `post-conversation.html` — All four show `"datePublished": "2025-04-01"` and `"dateModified": "2025-04-01"` in their JSON-LD, but the sitemap records `lastmod: 2026-05-16` for these files (they were updated in the most recent commit). Google uses `dateModified` as a freshness signal. Update `dateModified` to match the actual revision date; for posts significantly updated, consider also updating `datePublished` to the version date if the content has substantially changed.

- **Blog index (`blog.html`) missing BreadcrumbList schema** — `blog.html` — All five individual post pages correctly include a `BreadcrumbList` (Home → Blog → Post title). The blog listing page itself has no breadcrumb schema, so its SERP entry won't show the breadcrumb trail. Add: `Home → Blog` as a two-item `BreadcrumbList` pointing to `https://clearhead.in/` and `https://clearhead.in/blog`.

## What's already strong

- **Heading hierarchy is clean across all pages** — single H1 per page, H2/H3 nesting is logical, no skipped levels anywhere.
- **All images have meaningful alt text** — every `<img>` tag carries a descriptive `alt`; none are empty or missing.
- **No render-blocking scripts in `<head>`** — Cal.com embed, Razorpay checkout, and the Razorpay handler all load in `<body>` with no blocking in the document head.
- **All images under 500 KB** — largest is `coaching-mid.jpg` at 264 KB; hero image (`coaching-early.jpg`) is 252 KB. No page-weight risk.
- **robots.txt is correct** — `/quiz.html`, `/thank-you.html`, `/api/`, and `/.netlify/` are all disallowed; Sitemap URL is declared.
- **Local SEO is comprehensive** — `LocalBusiness` schema has complete NAP (name, address, phone), `geo` coordinates, `hasMap`, and `openingHours`; geo meta tags (`geo.region: IN-MH`, `geo.placename`) are present; Google Maps embed and "Open in Google Maps" link reinforce the physical location.
- **Homepage structured data is thorough** — `LocalBusiness`, `Person`, `ProfessionalService`, and `FAQPage` schemas are all present and cross-linked with `@id` references, which is the correct approach.
- **OG and Twitter card meta present on all content pages** — `og:type`, `og:title`, `og:description`, `og:image`, `og:url`, `og:locale`, `og:site_name`, `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image` are all set.
- **Canonical present on every page** — no page is missing a canonical tag.
