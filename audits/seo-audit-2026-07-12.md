## Low (3)
- **robots.txt Disallow conflicts with noindex** — `robots.txt:3-4` — `/thank-you.html` and `/quiz.html` are both Disallowed and carry `noindex`. Disallow blocks crawling, so Google can never see the noindex tag; if either URL earns an external link it can surface as "Indexed, though blocked by robots.txt" with no snippet. Fix: delete the two Disallow lines and rely on the existing `noindex, nofollow` meta tag (keep `/api/` and `/.netlify/` Disallows). *Unchanged since 2026-07-09.*
- **Duplicate partial business entity in homepage schema** — `index.html:74-97` — the `ProfessionalService` node (`#service`) has only `name`, `provider`, `serviceType`, `areaServed` — no address, geo, telephone, or price data — while the separate `LocalBusiness` node (`#business`) is fully populated. The two entities aren't cross-referenced beyond both pointing at `#vaibhav` as provider/owner. Fix: merge `ProfessionalService` into `LocalBusiness` as an additional `@type`, or add an explicit `@id` link between them. *Unchanged since 2026-07-09.*
- **External WhatsApp links missing `rel="noreferrer"`** — `ai-relevance.html:269,308`, `readiness.html:312,360` — four "Book a discovery call" links to `wa.me` use `target="_blank" rel="noopener"` but drop `noreferrer` (present elsewhere on the site, e.g. `index.html:518`). Not an indexing or ranking issue — `noopener` already covers the security-relevant part — but inconsistent with the rest of the site's convention. Fix: append `noreferrer` to the four `rel` attributes. (The same-page `Privacy Policy`/`Terms` links using `rel="noopener"` only are same-origin and don't need `noreferrer`.)
