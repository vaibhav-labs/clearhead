## Medium (2)

- **Missing clean URL redirect: `/pricing`** — `netlify.toml` — Every other indexed page (blog, runway, post-ai, etc.) has a `301 from /slug to /slug.html` redirect. `pricing.html` does not. Netlify Pretty URLs serves both `/pricing` and `/pricing.html` as 200 with the same content; the canonical tag mitigates duplicate-content risk but a 301 is the consistent, cleaner signal to Googlebot. Add: `[[redirects]] from = "/pricing" to = "/pricing.html" status = 301`

- **Missing clean URL redirect: `/post-new-city-loneliness`** — `netlify.toml` — Same issue as above for the newest blog post. Add: `[[redirects]] from = "/post-new-city-loneliness" to = "/post-new-city-loneliness.html" status = 301`

## Low (3)

- **Meta description 2 chars over limit** — `career-friction.html:8` — Description is 162 chars; target is ≤160. Trim by 2 chars. Current: "…and see what staying costs." (the trailing sentence is the easiest to shorten slightly).

- **OG image uses external Unsplash URL** — `post-ai-loneliness.html:17`, `post-new-city-loneliness.html:17` — Both posts use Unsplash CDN URLs as `og:image` rather than a self-hosted image. If Unsplash changes URL structure or rate-limits the referrer, social card previews break silently. The fix is to download the image and host it at `clearhead.in/` (as `VJ.jpg` and the three coaching images already are). Lower priority because Unsplash has been stable for years, but worth fixing before any active link-sharing campaign.

- **Orphaned `<h3>` in runway section** — `index.html:247` — The runway widget section uses `<h3 id="runway-h">` directly under a `<section>` without a containing `<h2>`. Every adjacent section uses `<h2>` as the section heading. This is a minor heading-hierarchy inconsistency. Screen readers and some crawlers expect h3 to be subordinate to an h2. Fix: promote to `<h2>` (which would also make it visually stronger as a section headline) or add a visually-hidden `<h2>` above it.
